## Findings

### 🔴 Red — Critical / blocking

1. **portability and persistence / corroborated — the universal journal publisher depends on POSIX-only durability operations**
   - **Sources**: Codex standard P1 and Codex adversarial Red; main-agent source verification.
   - **Location**: plan `10354-10383`, `5050-5057`, and `26494`; current
     `scripts/lib/atomic-write.mjs:14-34`.
   - **Evidence**: `syncDirectory` unconditionally opens and `fsync`s a directory, while both the
     journal and genesis publishers use raw `renameSync`. The required native Windows path cannot rely
     on directory file-descriptor fsync, and the new publishers bypass the repository's existing
     bounded `renameAtomicWithRetry` handling for Windows `EACCES|EPERM|EBUSY` sharing failures.
     Because every post-genesis mutation is routed through this publisher, the first initialization or
     mutation can fail on a supported Windows cell.
   - **Required response**: define one platform-aware durable-replace contract, reuse bounded Windows
     rename retry semantics, explicitly define the best available Windows flush guarantee, and execute
     real genesis plus journal commits on `windows-latest` rather than testing only path normalization.

### 🟡 Yellow — Concerns requiring response

1. **fail-closed recovery / solo 1-of-2 — staged event bytes authenticate themselves before canonical mutation**
   - **Source**: Codex adversarial Red; independently verified.
   - **Location**: plan `10487-10536`.
   - **Evidence**: the initial `assertSnapshotBytes` overwrites `marker.after.events_bytes` and
     `events_digest` with values recomputed from `stagedEvents`, so a same-length corrupted event stage
     passes that check. Recovery can then truncate/append events and replace state/hash before the final
     whole-candidate comparison notices the mismatch. This violates the promised corruption refusal
     and byte-identical no-repair behavior.
   - **Required response**: authenticate `beforeEvents + stagedEvents` against the immutable marker
     after-image before any unlink/truncate/append/replace, or bind an explicit suffix length/digest;
     add a same-length stage corruption fixture proving every canonical byte remains unchanged.

2. **symlink safety / solo 1-of-2 — dangling journal symlinks bypass the planned corruption guard**
   - **Source**: Codex standard P1; independently verified with a local filesystem probe.
   - **Location**: plan `10364-10368` and `10422-10428`.
   - **Evidence**: `durableUnlink` returns when `existsSync(path)` is false. For a dangling
     `.anchored-*` or canonical `.replace` symlink, `existsSync` follows the missing target and returns
     false even though `lstatSync(path).isSymbolicLink()` is true. Cleanup therefore skips the promised
     fail-closed rejection and may proceed to replace or reuse the pathname.
   - **Required response**: use `lstat` with explicit `ENOENT` handling as the existence test and reject
     every symlink before cleanup or publication; retain negative fixtures for dangling and live links.

3. **test executability / solo 1-of-2 — later crash-worker extensions reference exports they never import**

     import changes. The spawned child reaches `ReferenceError` rather than the injected exit 91.
   - **Required response**: include literal import diffs for every extension-owned symbol and make the
     embedded validator parse/check the final composed worker module.
