## Findings

### 🔴 Red — Critical / blocking

1. **recovery / verified — journal gateway calls helpers that have no executable implementation**
   - **Source**: Codex standard P1.
   - **Location**: plan `10217-10220` and Task 7B Step 3.
   - **Evidence**: `readAnchoredMarkerUnderLock`, `recoverAnchoredTransactionUnderLock`,
     `assertNoAnchoredMarkerUnderLock`, `publishAnchoredCandidateUnderLock`, and
     `readCurrentRunIdUnderLock` are called by the literal mutation gateway but are only promised in
     prose. The displayed code cannot be applied or executed as a complete recovery implementation.
   - **Required response**: provide literal, coherent journal helper implementations in Task 7B and
     make the embedded validator require their definitions.

2. **idempotency / corroborated — same caller and operation can recover a different business request**
   - **Source**: Codex standard P1 and Codex adversarial Red.
   - **Location**: plan `10314-10320`, `14174-14176`, `15956-16049`.
   - **Evidence**: the generic intent digest omits receipt/observation and other operation-specific
     behavior inputs. A retry with a different thread receipt or host observation can recover and
     publish the original pending candidate before rejecting the new request.
   - **Required response**: bind every request-distinguishing value into a bounded raw-free,
     domain-separated operation projection and add same-operation/different-input crash tests.

3. **locking / verified — Task 8A retains a non-reentrant lock across callbacks and external work**
   - **Source**: Codex adversarial Red.
   - **Location**: plan `10328-10331`, `13846-13852`, `13876-13882`, `14177-14486`;
     current handoff body `scripts/lib/handoff.mjs:119-185`.
   - **Evidence**: a single outer mutation context surrounds descriptor callbacks, a recursive emit
     race hook, Desktop probing, and artifact writes. The displayed tests deliberately re-enter the
     same non-reentrant run lock and would return `LOCK_BUSY` instead of the asserted outcome.
   - **Required response**: use short reserve and final-emit contexts sharing one public-operation
     intent; release the lock before callbacks, probes, file writes, spawning, sleeping, or hooks.

4. **recovery / verified — respawn and mutating CLI handlers pre-read before journal recovery**
   - **Source**: Codex adversarial Red.
   - **Location**: plan `10063-10068`, `20234-20258`, `21028-21054`.
   - **Evidence**: public `respawn` begins with `readRespawnAuthority`, and the CLI performs
     `requireLease` through the marker-rejecting reader before invoking the mutation. An exact retry
     after a respawn phase/settlement crash never reaches the recovery-aware gateway.
   - **Required response**: parse mutation arguments without a canonical pre-read; put each respawn
     transactional phase behind a recovery-aware context, releasing it before external spawn.

### 🟡 Yellow — Concern

1. **terminal cleanup — `releaseLease` omits the terminal carve-out**
   - **Source**: Codex standard P1.
   - **Location**: plan `13387-13390` and later Task 10B/10D terminal regressions.
   - **Evidence**: migrating release to the generic context calls anchored append without
     `allowTerminal: true`, so intentional no-live-binding cleanup of a completed/stopped run is
     rejected before its documented terminal semantics can run.
   - **Required response**: grant the narrow terminal/no-live-App cleanup option only after the
     in-lock fence and live-binding check, and test both permitted and rejected terminal cases.

2. **test executability — Task 7G and Task 10D call undefined crash helpers**
   - **Source**: Codex adversarial Yellow.
   - **Location**: plan `13327-13342`, `18452-18480`, embedded validator `26224-26230`.
   - **Evidence**: `assertRemainingPublisherCrash7g` and
     `assertPublicMutationCrashRecovery10d` are invoked but never defined. Prose promises pass the
     validator while the displayed tests fail with `ReferenceError`.
   - **Required response**: provide literal helper and worker dispatch implementations and validate
     the named definitions plus closed operation cases.

3. **schema fixture — a later initialized fixture uses a one-key projection**
   - **Source**: Codex adversarial Yellow.
   - **Location**: plan `8141-8149`.
   - **Evidence**: Task 7A initializes `request_projection: { marker: 'original' }` after Task 3B's
     exact 12-key schema is active.
   - **Required response**: use an exact valid projection and mutate a legitimate field for the
     paired-state test.

4. **privacy — abandoned and revoked failure events still accept extra raw fields**
   - **Source**: Codex adversarial Yellow.
   - **Location**: plan `8730-8745`, `8922-8972`, `9209-9231`.
   - **Evidence**: exact failure key checks run only when continuation phase is `failed`; hash-valid
     `app-task-abandoned` and `app-task-consent-revoked` events can retain an extra raw receipt field.
   - **Required response**: centralize exact keys for every failure/control subtype and add one
     hash-valid extra-field negative fixture per subtype.

5. **operation boundary — later public mutators still need a complete pre-read inventory**
   - **Source**: synthesis of the adversarial respawn finding and verified plan search.
   - **Location**: Task 7E/7F public mutators and Task 11B/CLI wiring.
   - **Evidence**: Round 3 named the App mutations, but the same marker-rejecting public pre-read
     pattern remains reachable in profile, acknowledgement, insights, review, workspace/detect, and
     respawn paths unless the migration inventory is exhaustive.
   - **Required response**: enumerate every mutating export/CLI, remove every pre-gateway canonical
     read, and prove the closed inventory in executable tests and the validator.
