### 🟡 Yellow — Concern

1. **security / partial — confirmed opaque receipt is outside the cross-log proof**
   - **Source**: Codex standard only (P2)
   - **Location**: `docs/superpowers/specs/2026-07-13-codex-app-native-task-continuation-design.md:390`
   - **Evidence**: `app-task-confirmed` is correlated only by attempt, child, and timestamp. A hash-valid rewrite can change `continuation.thread_id`, recompute `.loop.hash`, and retain a valid log; `already-confirmed`/`already-acquired` then no longer proves the original host receipt. The same gap applies to `unconfirmed_thread_id` retained by message-failure paths.
   - **Required response**: add one-way receipt digests to the fixed confirmation/failure events, verify them against the durable raw IDs, and add paired state/hash rewrite tests without exposing raw IDs in events or output.

2. **architecture / partial — paired projection+digest rewrite bypasses the immutable genesis proof**
   - **Source**: Codex adversarial only (reviewer severity Red)
   - **Location**: `docs/superpowers/plans/2026-07-13-codex-app-native-task-continuation.md:2530-2547,9663-9680`
   - **Evidence**: schema validation accepts any non-array `request_projection`; the common verifier only hashes it and compares the colocated digest. An attacker can change `request_projection.goal`, add deep/unbounded keys, recompute both request and state hashes, and leave `loop.goal` unchanged while all current checks pass.
   - **Required response**: implement exact key/type/depth/count/byte validation and a genesis semantic baseline. Immutable current facts must match directly; legitimately mutable facts must be derived from the stored genesis projection through their exactly-one anchored transition histories. Test paired rewrites and oversized/deep projections.

3. **error-handling / partial — journal recovery is unreachable in the crash states and caller identity is optional**
   - **Source**: Codex adversarial only (reviewer severity Red)
   - **Location**: `docs/superpowers/plans/2026-07-13-codex-app-native-task-continuation.md:9721-9821`
   - **Evidence**: `appendAnchored` reads/verifies state/log before the publisher, and `commitVerifiedEventsUnderLock` verifies again before its publisher call. Partial suffix or state-before-hash crashes therefore fail before recovery can run. The same example silently falls back from missing `opts.callerBinding` to the current lease even though prose calls caller binding mandatory, allowing recovery identity to be inferred before the caller fence.
   - **Required response**: at the first run lock, require explicit pre-state binding, compare a pending marker before any repair, refuse foreign binding with the established fence/no-write result, recover exact binding, then reread/reverify and restart the API. Remove the current-lease fallback and test every crash point.

4. **architecture / partial — legacy lineage checkpoint has no implementation in the common gateway**
   - **Source**: Codex adversarial only
   - **Location**: `docs/superpowers/plans/2026-07-13-codex-app-native-task-continuation.md:9721-9749`
   - **Evidence**: Task 7B promises automatic `lease-lineage-baselined` insertion and tests a generation-3 migration, but the complete gateway only appends caller-supplied `eventSpecs`. It has no eligibility check or checkpoint prepend, so the advertised migration cannot occur.
   - **Required response**: add a closed eligibility helper and automatic prepend after explicit fence/current/root/hash/legacy checks, bind the checkpoint to the verified pre-state hash and current generation, and have the generation-3 test exercise this code.

5. **architecture / partial — Task 8A restores raw writers after Task 7G closes them**
   - **Source**: Codex adversarial only
   - **Location**: `docs/superpowers/plans/2026-07-13-codex-app-native-task-continuation.md:13352,13380`
   - **Evidence**: Task 7G routes every post-genesis lease/breaker mutation through the journal, but later Task 8A retains/adds raw `writeState` calls in reservation and rollback. Sequential execution either cannot apply cleanly or reopens the event/state/hash crash window, and Task 8A does not rerun the publisher-closure test.
   - **Required response**: rewrite Task 8A against the post-7G journalized lease APIs, preserve the fixed reserve/rollback events and explicit caller binding, and rerun closure after Task 8A and in final audits.

6. **test-coverage / partial — the embedded validator contradicts the claimed 46-task PASS**
   - **Source**: Codex adversarial only
   - **Location**: `docs/superpowers/plans/2026-07-13-codex-app-native-task-continuation.md:24956-25009`; `docs/handoff/2026-07-13-codex-app-native-task-continuation-evidence.md:226-229`

Running the embedded equality logic yields `headingCount=46`, `expectedCount=45`, `missingFromValidator=["7G"]`; the recorded PASS came from an external simplified validator rather than the plan's own validator.
   - **Required response**: add 7G to the inventory, coverage rows, anchor families, and embedded `expectedTasks`; run the exact embedded validator and correct the evidence language to identify the exact command that passed.
