## Findings

### 🔴 Red — Blocker

1. **Anchored inline request omits the actual episode task / 2-of-2**
   - **Sources**: Codex standard P1; Codex adversarial Red 1; independently reproduced.
   - **Evidence**: `episodeRequestMarkdown` writes only the literal placeholder
     `Execution plane: fill the maker/checker task here` and does not consume a bounded task or the
     anchored contract (`plan:9152-9175`). Final tests require that placeholder (`plan:15751-15770`).
     The design simultaneously promises a self-contained request whose task-specific enrichment is
     fixed before creation (`design:741-755`). With request files removed, a fresh execution plane
     cannot reconstruct what work to perform from the anchored request.
   - **Required response**: accept one bounded task payload before episode creation and bind its
     exact text into the immutable projection, canonical Markdown, digest, object, and event. Include
     the contract or its complete task-bearing projection so a fresh maker/checker needs no prior
     conversation context. Add fresh/replay/tamper/CLI regressions.

### 🟡 Yellow — Required concerns

1. **Review dispatch does not return the replacement inline request / solo 1-of-2**
   - **Source**: Codex standard P1; independently reproduced.
   - **Evidence**: same-ID replay returns only `{checkerEpisodeId, reviewer, descriptor}`
     (`plan:17504-17510`). The fresh path verifies that `checker.request_markdown` exists but its
     public response/CLI path does not add the Markdown or digest (`plan:17673-17686,18368-18377`).
     This contradicts the design promise that both `episode new` and `review dispatch` return the
     verified inline request (`design:749-755`).
   - **Required response**: return the exact durable `request_markdown` and digest on both fresh and
     replay paths, preserve one exact public/CLI shape, and assert byte-identical retry behavior.

2. **Crash-capability closure is token-based rather than capability-complete / solo 1-of-2**
   - **Source**: Codex adversarial Red 2; independently reproduced as a contract/validator gap.
   - **Evidence**: final public mutation afterimages retain `beforeFinalAppendFn` and
     `deps.beforeAppendFn` callbacks (`plan:20935-20940,21258-21261,30440-30445,30484-30488`), and
     Task 11C's worker constructs/passes them (`plan:29591-29619`). These are documented process-race
     test seams and execute outside the anchored callback, so this is not evidence of an
     execution-plane CLI injection. However, the design claims no public mutation/helper worker
     carries a function-valued crash capability (`design:757-760`), while the validator rejects only
     the literal `crashProbe` token and inventories later source surfaces without installing them
     (`plan:36881-36959`). A renamed callback can therefore evade the claimed closure.
   - **Required response**: either replace process-race barriers with a private scalar/protocol that
     cannot execute arbitrary caller code, or narrowly specify and mechanically enforce the exact
     allowed outside-lock test seams while separately proving none can enter the journal publisher.
     The validator must reject capability-equivalent crash callbacks, not one spelling.

3. **Malformed episode creation identities bypass the global uniqueness map / solo 1-of-2**
   - **Source**: Codex adversarial Red 3; independently reproduced.
   - **Evidence**: episode verification reports ambiguous caller/dispatch presence but adds an
     identity to the global map only when the derived string ends in lowercase 64-hex; a malformed
     non-null digest receives no error (`plan:11194-11208`). The workstream verifier correctly
     rejects a malformed digest in its projection check (`plan:11260-11286`). Two distinct versioned
     episode objects/events can therefore use the same malformed non-hex identity and evade global
     duplicate detection if the remaining state/event/proof bytes are recomputed.
   - **Required response**: require exactly one well-formed 64-hex caller or dispatch identity before
     map insertion, reject malformed event/object identity symmetrically, and add distinct-episode
     duplicate plus malformed caller/dispatch regressions.
