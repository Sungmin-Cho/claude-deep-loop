## Findings

### 🔴 Red

1. **A swapped parent directory redirects the pathname write before validation can fail**
   - **Sources**: standard P2 and adversarial Red; independently reproduced by main.
   - **Evidence**: Gate 8 component-wise `lstat`s a parent and later opens a final pathname with
     no-follow/exclusive flags. Final-component `O_NOFOLLOW` does not pin ancestors
     (`docs/superpowers/plans/2026-07-13-codex-app-native-task-continuation.md:35866-35876`).
   - **Counterexample**: after the parent check, rename the reports directory and replace it with a
     symlink to an unrelated directory whose final basename is absent. The exclusive pathname open
     follows the swapped ancestor and writes/fsyncs outside the canonical root. A later identity check
     can report failure but cannot undo the out-of-root write.
   - **Required response**: do not materialize the authoritative report/response through worktree
     pathnames. Store exact Buffers directly as Git blobs and construct the receipt tree/commit from
     their object IDs. If a worktree copy is desired later, it is non-authoritative and must use Git's
     checkout machinery under the separately verified closeout lifecycle.

### 🟡 Yellow

1. **Final revalidation ends before `git add`, allowing unreviewed bytes into the receipt commit**
   - **Sources**: standard P2 and adversarial Yellow; independently reproduced by main.
   - **Evidence**: Gate 8 revalidates paths/bytes and only afterward mutates evidence/bundle, force-adds
     four paths, checks the name set, and commits. It never compares staged or committed blob IDs to
     the retained bytes (`...plan.md:35882-35893`).
   - **Counterexample**: replace ignored destination report A with sanitized B after final revalidation
     but before `git add`. Git stages B; the four-path name set and `diff --check` still pass while the
     evidence records A's hash.
   - **Required response**: hash the four exact Buffers into Git objects, populate a private index with
     those exact OIDs, verify the tree entries and exact four-path delta, create the commit from that
     tree, and update the closeout branch only with an expected-parent atomic ref CAS. Verify the final
     commit tree contains the same four OIDs; never use live worktree bytes as staging input.

2. **Process-object continuity is not a mechanically meaningful receipt authority**
   - **Source**: adversarial Yellow; independently analyzed by main.
   - **Evidence**: the plan requires one uninterrupted process to retain a Buffer but cannot distinguish
     an equal reconstructed Buffer after a restart or prove JavaScript Buffer immutability. Report and
     Respond also complete at different times (`...plan.md:35854-35865`).
   - **Required response**: replace process identity with content identity. A deterministic sanitized
     byte sequence is authoritative by SHA-256 plus Git blob OID; byte-identical reconstruction yields
     the same OID, while any mutation yields a different OID. Interruptions before the expected-parent
     ref CAS leave no receipt commit; recovery revalidates exact inputs/objects rather than claiming
     uninterrupted object identity.
