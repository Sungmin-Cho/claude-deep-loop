1. **The verified `episodes` parent can still redirect the first staging write / 2-of-2**
   - **Sources**: Codex standard P1; Codex adversarial Red.
   - **Evidence**: `publishDirectoryFileDurably` validates the parent with `lstat`, then later calls
     pathname-based `mkdir(staging)` and writes `stagedFile`. Replacing the `episodes` parent with a
     symlink/junction between those operations redirects the private staging directory and
     `request.md` outside `.deep-loop`. The added race swaps only the child episode path after
     staging already exists, so it cannot detect this parent swap.
   - **Required response**: bind staging creation, file write, and final publication to one verified
     non-followed parent identity, or fail closed on hosts without the required primitive. Add
     parent swaps immediately before first staging creation and immediately before publication,
     proving external bytes and directory inventories remain unchanged on POSIX and Windows.

2. **The legacy checkpoint does not authenticate proof-bearing pre-state / 2-of-2**
   - **Sources**: Codex standard P1; Codex adversarial Yellow; independently reproduced.
   - **Evidence**: `legacyEpisodeBaselineDigest` omits episode status, artifacts/result/review proof,
     workstream status, and related finish-authority fields. Separately, the verifier accepts any
     well-formed 64-hex `pre_state_digest` without binding it to durable state or an anchored event.
     A hash-valid rewrite can therefore change legacy makers/checkers/workstreams to proof-satisfying
     terminal values while retaining a verifier-accepted checkpoint and passing finish proof.
   - **Required response**: define one canonical proof-bearing legacy pre-state projection, bind its
     digest to the checkpoint transaction, and verify every later proof-bearing change through exact
     transition events. Test status/artifact/review/workstream rewrites and a substituted well-formed
     pre-state digest under an otherwise valid recomputed chain.
