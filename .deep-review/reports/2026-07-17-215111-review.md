## Findings

### 🔴 Red

1. **The four-buffer receipt requires values derived from the same buffers**
   - **Sources**: standard P1 and adversarial Red; independently reproduced by main.
   - **Evidence**: Gate 8 first defines the evidence, bundle, report, and response Buffers, then derives
     their blob IDs, private tree, commit, and CAS result, but later requires evidence and bundle to
     record all four hashes/object IDs, tree ID, commit ID, and CAS result
     (`docs/superpowers/plans/2026-07-13-codex-app-native-task-continuation.md:35854-35866,35881-35899,35911-35914`).
   - **Counterexample**: inserting an evidence Buffer's first SHA-256 into that Buffer changes the
     SHA-256. Tree and commit IDs therefore also change; a successful CAS result cannot exist until
     after the commit is published. The procedure provides no finite fixed-point construction.
   - **Required response**: split publication into a finite payload commit A and an attestation commit
     B. A contains the exact four payload paths without self-derived values. B changes only
     evidence/bundle and records A's four hashes/OIDs, tree, commit, and CAS result. B must not claim
     its own derived IDs or CAS result; the fully qualified branch ref and independent verification
     bind B externally. Preserve an exact cumulative four-path delta from the pre-receipt parent.

2. **Inherited Git environment/config can publish a branch whose commit exists only externally**
   - **Source**: adversarial Red; independently reproduced by main.
   - **Evidence**: the helper invokes Git plumbing but does not yet specify a canonical executable,
     allowlisted environment, canonical git/common/object directories, replacement/alternate guards,
     disabled hooks, or a fully qualified non-symbolic ref
     (`...plan.md:35868-35899`).
   - **Counterexample**: set `GIT_OBJECT_DIRECTORY` to an external directory and
     `GIT_ALTERNATE_OBJECT_DIRECTORIES` to the real store. Blob/tree/commit objects are written
     externally, while `update-ref` publishes the commit ID to the real repository. With the injected
     environment removed, the branch references a missing commit. A configured
     `reference-transaction` hook also runs during `update-ref`.
   - **Required response**: bind one canonical Git executable and digest; construct a minimal
     allowlisted child environment; explicitly bind canonical git/common/object directories and the
     private index; reject alternates, replace refs, symlinked control paths, unexpected repository
     format/config, and Git injection variables; disable replacements and hooks; require a fully
     qualified validated non-symbolic branch ref; and prove all objects are readable from the canonical
     store before and after CAS.
