## Findings

### 🟡 Yellow — Required concerns

1. **Task 6A's validator accepts Windows-breaking `.pathname` process arguments**
   - **Source**: Codex adversarial; independently reproduced.
   - **Evidence**: the corrected Task 6A launches use `FIXED_INIT_CRASH_WORKER`, but the validator
     rejects only the case-sensitive literal pattern `/worker\.pathname\b/u` (`plan:36125-36131`).
     Replacing one launch argument with `FIXED_INIT_CRASH_WORKER_URL.pathname` or inline
     `new URL(...).pathname` preserves all required tokens while bypassing that pattern. The
     independent mutation probe confirmed the validator predicate remains false for the bypass.
   - **Required response**: reject any `.pathname` use in the Task 6A card, require the native
     filesystem worker string at every launch, and add a validator self-mutation regression for an
     uppercase/inline URL pathname argument.

2. **Complete genesis fixtures discard the event bound by `loop.event_log_head`**
   - **Source**: Codex standard P1; independently reproduced.
   - **Evidence**: `validGenesis` returns only `JSON.stringify(buildCanonicalGenesis(...).loop)`
     (`plan:3578-3580`) even though `buildCanonicalGenesis` also returns `genesisEvents` and binds
     the generated `run-initialized` event into `loop.event_log_head`. All four callers write only
     `loop.json` and `.loop.hash`, so `strictIntegrity` sees an empty event log and rejects the
     supposedly complete fixture before the Task 4B/4C assertions.
   - **Required response**: return both loop bytes and serialized genesis event-log bytes, write
     both in every complete fixture, and make the validator prove the four event-log writes and
     strict-integrity consistency.

3. **The final episode replacement removes the `maker|checker` role allowlist**
   - **Source**: Codex standard P1; independently reproduced against current production source.
   - **Evidence**: the displayed final `createEpisode` replacement checks only that `role` is a
     nonempty string (`plan:18367-18369`), while current `scripts/lib/episode.mjs` rejects roles
     outside `maker` and `checker`. Following the plan would persist a malformed role that normal
     routing/proof logic ignores.
   - **Required response**: preserve the exact role allowlist before any durable mutation, add
     direct invalid-role regressions, and require the allowlist in the composed final afterimage.

4. **The same-ID retry generates a different request digest**
   - **Source**: Codex standard P2; independently reproduced.
   - **Evidence**: `repeatedInput` omits `task` (`plan:16459-16460`), while `episode7f` synthesizes a
     task from an incrementing counter. The independent probe produced
     `Execute reviewer fixture episode 1.` then `Execute reviewer fixture episode 3.` for the
     retry, so the digest conflicts instead of returning the original episode.
   - **Required response**: include a stable explicit task in the repeated input and add a
     validator/probe assertion that exact same-ID retry uses byte-identical request identity.

5. **The final episode replacement removes absolute and traversal artifact-path rejection**
   - **Source**: Codex standard P2; independently reproduced against current production source.
   - **Evidence**: the displayed replacement validates only that `expectedArtifacts` contains
     strings (`plan:18371-18375`), while current `scripts/lib/episode.mjs` rejects absolute paths and
     `..` segments. The replacement could durably create an episode whose later completion is
     rejected as `EPISODE_ARTIFACT_ESCAPE`.
   - **Required response**: preserve both path-safety guards before mutation, add relative-parent
     and absolute-path regressions, and require those guards in the composed final afterimage.
